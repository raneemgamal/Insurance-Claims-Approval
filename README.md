# Insurance Claims Analysis and Prediction

## Project Overview

This project analyzes an insurance claims dataset and builds machine learning models to understand customer/vehicle patterns and predict claim approval status. The work is implemented in Jupyter notebooks and includes data preparation, exploratory data analysis, preprocessing, clustering, classification, model evaluation, and model saving.

The main target variable is:

```text
claim_status
```

Where:

- `0` = claim not approved
- `1` = claim approved

## Project Files

| File | Description |
|---|---|
| `insurance_claims.ipynb` | Main classification notebook. It performs EDA, preprocessing, trains Logistic Regression, Random Forest, and XGBoost, compares models, and saves the best model. |
| `insurance_claims_2.ipynb` | Complete assignment-style notebook. It includes data understanding, data cleaning, EDA, preprocessing, clustering, classification, optional extra model comparison, and optional regression. |
| `Insurance claims data.csv` | Dataset file required to run the notebooks. |
| `insurance_claims_model.pkl` | Saved trained model generated by `insurance_claims.ipynb`. |

## Dataset Description

The dataset contains insurance policy, customer, vehicle, and claim-related information. It includes both numerical and categorical features.

Examples of features used in the project include:

- `policy_id`
- `subscription_length`
- `vehicle_age`
- `customer_age`
- `region_code`
- `region_density`
- `segment`
- `model`
- `fuel_type`
- `engine_type`
- `airbags`
- `transmission_type`
- `ncap_rating`
- `claim_status`

The dataset has 41 columns, and the target column is `claim_status`.

## Objectives

The main goals of the project are:

1. Understand the structure and quality of the insurance claims data.
2. Clean and preprocess the dataset for machine learning.
3. Explore relationships between vehicle, customer, and policy features.
4. Handle class imbalance in the target variable.
5. Apply unsupervised learning using clustering.
6. Build supervised classification models to predict claim status.
7. Evaluate models using suitable performance metrics.
8. Save the best trained model for future use.

## Project Workflow

### 1. Data Understanding

The notebooks start by loading the dataset and checking:

- Dataset shape
- Column names
- Data types
- Summary statistics
- Number of unique values
- Target class distribution
- Numerical and categorical feature types

### 2. Data Cleaning

The cleaning stage includes:

- Checking missing values
- Checking duplicated rows
- Dropping duplicates if found

- Checking invalid values

### 3. Exploratory Data Analysis

EDA is used to better understand the dataset through visualizations such as:

- Target distribution count plot

The preprocessing stage prepares the data for machine learning by applying:

- Evaluate clusters using:

## Important Notes About the Results

The dataset is highly imbalanced, meaning most records belong to class `0` and fewer records belong to class `1`. Because of this, accuracy alone is not enough to judge model performance.

For this project, ROC-AUC, recall, and F1-score are important because the goal is to detect approved claims correctly, especially when the positive class is rare.

Random Forest achieved high accuracy, but this is mainly because it predicted the majority class well. Logistic Regression gave better balanced performance on the minority class.
